# 项目说明：无人机物体箱定位系统（Flask端云）

## 1. 项目定位
这是一个“无人机端 + 服务器端”的实时监控与数据上传系统，用于：
- 无人机侧进行条码/二维码检测与 GPS 采集
- 通过网络将检测结果与定位数据传输到服务器
- 服务器端提供 Web 界面与 API，用于实时监控与数据查看

## 2. 当前状态
- 项目结构清晰：无人机端、服务器端、测试、脚本、文档分离。
- Windows 测试与 Linux 部署均有说明文档。
- 注意：主 README 中提到的 docs 文件列表与当前 docs 目录实际文件数量存在不一致，已在“文档索引”中如实说明。

## 3. 目录结构速览（关键内容）
- `drone_side/`：无人机端核心逻辑（摄像头、条码识别、GPS、传输）。
- `server_side/`：服务器端（Flask + SocketIO + SQLite + templates）。
- `scripts/`：部署与运维脚本（Linux 为主）。
- `tests/`：按平台/类型划分的测试工具与说明。
- `docs/`：部署/测试说明文档。

## 4. 快速运行
### 4.1 无人机端（开发/本机）
- `cd drone_side`
- `pip install -r requirements.txt`
- `python main.py`

### 4.2 服务器端（开发/本机）
- `cd server_side`
- `pip install -r requirements.txt`
- `python app.py`

> 具体端口、数据库路径与日志配置请见配置文件与 Linux 部署文档。

## 5. 依赖与环境
- Python 3.8+（文档示例中为 3.12 test 环境）
- 依赖分别在：
  - `drone_side/requirements.txt`
  - `server_side/requirements.txt`

## 6. 配置与密钥
- 本项目以“服务器地址/端口”等配置为主，建议通过环境变量或配置文件统一管理。
- Windows 测试文档提供了环境变量方式（便于在不同服务器间切换）。

## 7. 输出与数据
